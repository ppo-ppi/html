## 1. 레이아웃과 그리드 (Layout & Grid)

* **일관성이 핵심**: 디자인할 때는 한 칸을 기준으로 전체적인 일관성을 유지해야 함.
* **그리드 사이즈**: 보통 내부 알맹이는 **80%** 정도로 잡고, 너무 커지지 않게 `max-width`를 지정함. (예: 1080px)
    * 큰 화면에서는 1080px로 고정되지만, 작은 화면에서는 80% 비율을 유지하도록 설정.
* **중앙 정렬 공식**: 그리드 요소들을 가운데로 보낼 때는 `margin: 0 auto;`를 사용함.
* **높이 설정**: 
    * 박스 높이를 억지로 잡기보다 내부 알맹이부터 높이를 쌓아 올려서 컨텐츠가 커지면 부모 박스도 자연스럽게 커지게 만드는 것이 좋음.
    * 단, 텍스트가 두 줄로 떨어지는 경우 등 특정 디자인에서는 박스에 고정 높이가 필요함.

## 2. 단위와 계산식 (calc)

* **가로 vs 세로**: 가로는 비율(**%**)로, 세로는 고정값(**px**)으로 주는 것이 일반적임.
* **12그리드 공식**: `width: calc(100% / 12 * 칸수);`를 활용하면 직관적으로 레이아웃을 짤 수 있음.
    * 요소 간 간격(Gutter)이 있다면 전체 너비에서 해당 값을 빼줌. (예: `width: calc(100% / 3 - 10px);`)
* **계산의 기준**: `100%`는 기본적으로 부모 요소의 너비를 의미함.

## 3. 정렬과 배치 (Flex & Alignment)

* **display: flex**: 블록 요소들을 가로로 나열하거나 부모 기준으로 정렬할 때 필수임. (ul 안의 li 나열 등)
    * `justify-content: space-between;`: 양 끝으로 쪼개서 자동 정렬.
    * `column-gap: 20px;`: 요소 사이의 간격을 직관적으로 벌릴 때 사용.
* **텍스트 정렬**: 
    * `text-align: center;`: 텍스트나 인라인 요소를 가로 중앙으로 보냄. (자식에게 물려줌)
    * `line-height`: 글씨를 세로 중앙에 맞출 때 사용. 박스 높이만큼 주면 정중앙에 옴.
    * **주의**: 텍스트가 두 줄일 때는 `line-height`를 박스 높이의 절반 정도로 줄여야 함.

## 4. 마진과 패딩 (Margin & Padding)

* **Margin (마진)**: 요소와 요소(형제) 사이를 떨어트릴 때 사용.
    * 주로 `margin-top`을 기준으로 작업하며, `margin-bottom`은 잘 안 씀.
    * 마진에 음수(`-`) 값을 주면 요소끼리 겹치게 할 수 있음.
* **Padding (패딩)**: 박스 내부에 살을 붙이거나 안쪽으로 밀어 넣을 때 사용.
    * 배경색이 있는 박스에서 내부 요소를 띄울 때 적합함.
    * 마진은 겹칠 수 있지만(병합 현상), 패딩은 겹치지 않음.
* **구분 팁**: 블록 안에서 요소를 밀어 넣을 땐 **패딩**, 컨텐츠 덩어리끼리 떨어트릴 땐 **마진**.

---

## 5. 텍스트와 스타일링 (Typography & Tips)

* **속성 상속**: 부모에게 값을 주면 자식에게도 영향을 줌. (예: 헤더에 `line-height`를 주면 내부 gnb에도 적용됨)
* **텍스트 제어**:
    * `text-transform`: `uppercase`(대문자), `capitalize`(첫 글자만 대문자).
    * 박스 너비가 있어야 텍스트 줄바꿈이 일어남.
    * 텍스트가 박스를 넘칠 때는 `overflow: hidden;`으로 처리.
* **요소 특징**: 
    * 버튼은 인라인이지만 예외적으로 `width`를 가질 수 있음.
    * 이미지나 블록 요소를 가운데 정렬하고 싶으면 `margin: 0 auto;`.
* **기타 팁**: 
    * 기본 CSS(Default CSS)가 있으므로 항상 리셋(Reset)하고 시작할 것.
    * VS Code에서 경로 써놓고 `Ctrl + 클릭`하면 파일 생성 가능해서 편함.
